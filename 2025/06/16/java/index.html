注解学习

@RestController 则是@Controller和@ResponseBody这两个注解的组合。它标识的类中的方法返回值会被自动转换为 JSON（或其他格式），而不是渲染视图。
@RequestMapping("/hello") @RequestMapping("/hello") 是 Spring MVC 中的一个注解，用于将 HTTP 请求映射到处理方法上。它可以用于类级别或方法级别，指定一个 URL 路径，当接收到该路径的请求时，Spring 会调用对应的处理方法。
@RequestMapping 是一个通用的注解，适用于不同的 HTTP 请求方法。
如果你只需要处理某一种 HTTP 方法（例如 GET 或 POST），可以使用更简化的注解，如 @GetMapping、@PostMapping 等。
@Autowired 是 Spring 的自动依赖注入注解，帮助你自动注入所需要的依赖对象。

标准结构

your-project-name/          # 项目根目录（命名建议：小写+横杠，如user-service）
├── pom.xml                 # Maven核心配置文件（依赖管理、打包方式等）
├── .gitignore              # Git忽略文件（过滤IDE配置、编译产物等，必加）
├── README.md               # 项目说明文档（功能、启动方式、依赖等，必加）
├── src/                    # 源代码根目录
│   ├── main/               # 主程序代码（生产环境代码）
│   │   ├── java/           # Java源代码根目录（必须与pom.xml的groupId对应）
│   │   │   └── com/        # 公司/组织域名反向（如com.ajmer）
│   │   │       └── ajmer/ # 项目/模块标识（如example、usercenter）
│   │   │           ├── controller/ # 控制层（如API接口、请求处理，如UserController）
│   │   │           ├── common/    # 公共
│   │   │           │   └── constant/   # 固定变量
│   │   │           │   └── enums/   # 枚举
│   │   │           ├── service/    # 服务层（业务逻辑，如UserService）
│   │   │           │   └── impl/   # 服务实现类（如UserServiceImpl）
│   │   │           ├── mapper/        # 数据访问层DAO/Mapper（数据库交互，如UserDao）
│   │   │           │   └── mapper.java/ # MyBatis等ORM的映射文件（如UserMapper.xml）
│   │   │           ├── model/      # 数据模型（实体类，如User、Order；DTO/VO也可放此目录）
│   │   │           │   ├── entity/ # 细分：数据库表对应实体（如UserEntity）
│   │   │           │   ├── query/  # 查询对象
│   │   │           │   ├── dto/    # 细分：数据传输对象（如UserDTO，前后端交互）
│   │   │           │   └── vo/     # 细分：视图对象（如UserVO，前端展示）
│   │   │           ├── config/     # 配置类（如Spring配置、线程池配置，如WebConfig）
│   │   │           ├── utils/       # 工具类（如日期工具、加密工具，如DateUtil）
│   │   │           ├── exception/  # 异常类（自定义异常、全局异常处理，如BizException）
│   │   │           └── YourMainApplication.java # 主启动类（Spring Boot项目必备）
│   │   └── resources/              # 主程序资源文件（编译后放入classpath）
│   │       ├── application.yml     # 核心配置文件（Spring Boot项目，替代properties）
│   │       ├── application-dev.yml # 开发环境配置（多环境隔离）
│   │       ├── application-prod.yml # 生产环境配置（多环境隔离）
│   │       ├── static/             # 静态资源（如JS、CSS、图片，Web项目用）
│   │       ├── mapper/                         ← MyBatis XML 目录
│   │       │   └── SysUserMapper.xml           ← 对应 Mapper 接口的 SQL 实现
│   │       ├── templates/          # 模板文件（如Thymeleaf、Freemarker模板，Web项目用）
│   │       └── META-INF/           # 元信息目录（如MANIFEST.MF、services配置）
│   └── test/                       # 测试代码目录（与main结构镜像，不影响生产环境）
│       ├── java/                   # 测试源代码根目录（包结构与main完全一致）
│       │   └── com/
│       │       └── example/
│       │           ├── controller/ # 控制层测试（如UserControllerTest）
│       │           ├── service/    # 服务层测试（如UserServiceTest）
│       │           └── dao/        # 数据访问层测试（如UserDaoTest）
│       └── resources/              # 测试资源文件（仅测试时生效，覆盖main资源）
│           └── application-test.yml # 测试环境配置（如测试数据库连接）
└── target/                         # Maven/Gradle编译产物目录（自动生成，无需手动维护）
    ├── classes/                    # 主程序编译后的class文件+资源文件
    ├── test-classes/               # 测试程序编译后的class文件+测试资源
    └── your-project-name-1.0.0.jar # 最终打包产物（JAR/WAR，名称由pom.xml配置）

              ↓ 实现（implements）
┌─────────────────────────┐
│  SysUserServiceImpl.java│
└─────────────┬───────────┘
              ↓ 依赖（注入Mapper）
┌─────────────────────────┐
│  数据访问层 (DAO/Mapper) │
│   SysUserMapper.java    │  ← 绑定（XML配置）
└─────────────┬───────────┘
              ↓ 关联（namespace+SQL）
┌─────────────────────────┐
│   SysUserMapper.xml     │
└─────────────────────────┘
# 以下是所有层的公共依赖（数据载体）
┌─────────────────────────┐
│   数据模型 (Entity)      │
│      SysUser.java       │  ← 被Controller、Service、Mapper依赖
└─────────────────────────┘
