服务部署

部署前准备

1.需要修改项目里或者打好的 jar 包里(路径:BOOT-INF/classes/application.yaml)的服务连接地址 2.填写正确的 mysql 连接方式(ip 地址:端口，用户名/密码，数据库实例名) 填写正确的 redis 连接方式(ip 地址:端口，密码) 填写正确的 minio 连接方式(ip 地址:端口，用户名/密码) 3.把打好的项目或者 jar 包上传到服务器，根据后端服务名上传到各个目录下即可单机环境:

#如打的是core.jar，上传到服务器/root下
mv /root/core.jar /data/stand-alone-deploy/saas/core/jar

集群环境:

#如打的是core.jar，上传到服务器/root下
mv /root/core.jar /data/jar/core

## 单机模式
1.需参考中间件部署把docker,docker-compose命令部署好
2.下载后端资源目录
        服务器能连互联网
        wget 45.119.60.33:8881/stand-alone-deploy
        wget 45.119.60.33:8881/jdk-17.tar

        服务器不能连互联网
        #后端资源目录下载链接，下载好后传在服务器上

        45.119.60.33:8881/stand-alone-deploy
        45.119.60.33:8881/jdk-17.tar
3.创建部署目录,并把后端资源目录放在其路径下,并解压镜像
```plaintext
mkdir /data
mv stand-alone-deploy /data
docker load -i jdk-17.tar

4.进入到工作目录，执行命令部署

#默认工作目录是/data/stand-alone-deploy
cd /data/stand-alone-deploy
docker-compose up -d

集群模式

1.需安装好 k8s 环境 2.下载部署脚本 服务器能连互联网 #后端更新镜像脚本,需下载到任意一台 node 节点 wget 45.119.60.33:8881/backend.sh #后端 jar 包部署目录,需下载到任意一台 node 节点 wget 45.119.60.33:8881/jar #后端服务更新脚本,需下载到任意一台 master 节点 wget 45.119.60.33:8881/hot_deploy.sh #后端 yaml 文件目录,需下载到任意一台 master 节点 wget 45.119.60.33:8881/script #前端更新脚本,需下载到 nginx 服务器上 wget 45.119.60.33:8881/frontend.sh #jdk 镜像 wget 45.119.60.33:8881/jdk-17.tar 服务器不能连互联网 #脚本下载链接，下载好之后传在服务器上 #需下载到任意一台 node 节点 45.119.60.33:8881/backend.sh #需下载到任意一台 node 节点 45.119.60.33:8881/jar #需下载到任意一台 master 节点 45.119.60.33:8881/hot_deploy.sh #需下载到任意一台 master 节点 45.119.60.33:8881/script #需下载到 nginx 服务器上 45.119.60.33:8881/frontend.sh #jdk 镜像 45.119.60.33:8881/jdk-17.tar 3.给后端脚本加上可执行权限(node 节点)，解压镜像并把后端部署目录放在工作目录下

chmod + backend.sh
docker load -i jdk-17.tar
mv jar /data/

4.执行脚本部署服务

bash backend.sh  #根据所部署的服务输入名称

5.返回 master 节点给脚本加上权限并执行更新

chmod +x hot_deploy.sh
bash hot_deploy.sh  #根据所部署的服务输入名称

6.给前端脚本加上可执行权限并部署服务

chmod +x frontend.sh
bash frontend.sh  #根据所部署的服务输入名称
